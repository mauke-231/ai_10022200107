ACity RAG System — Architecture Diagram

CS4241 Introduction to Artificial Intelligence · 2026 · Author: Maukewonge Yaw Nyarko-Tetteh | Index: 10022200107

Data Flow (Step-by-Step):
1. Raw PDFs and CSV are loaded → cleaned → chunked into 641 semantic chunks (Part A)
2. All chunks are embedded using all-MiniLM-L6-v2 (384 dimensions) and stored in FAISS + BM25 keyword index (Part B)
3. User query arrives via Streamlit UI → Memory Store is checked for relevant past exchanges (Part G)
4. Hybrid retrieval (vector + keyword) retrieves top-k candidates; failure detection checks score threshold (Part B)
5. Context selection deduplicates and token-budgets the chunks (Part C)
6. Prompt Builder (v1/v2/v3) injects context + memory + hallucination guardrails (Part C)
7. LLM Client sends prompt to chosen provider (Anthropic / OpenAI / Google) (Part D)
8. Response + retrieved chunks + scores shown in UI; Q&A stored in memory (Parts D + G)
9. All stages logged to JSONL for experiment tracking (Part D)
